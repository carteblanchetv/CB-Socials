Carte Blanche Socials Hub Help Guide

1. Posting Schedule Optimiser

This panel suggests the three best hours to post content for each social platform on the selected day of the week.

  • Benchmarks: Shows general public activity levels for your sector. Recommended to use for new campaigns or empty accounts.
  • My Analytics: Displays peak times tailored directly to your followers' historical interaction patterns, unlocked after importing analytics CSVs.
  • Time Pills: Click any suggested time pill (e.g. 09:30 AM) to instantly open a pre-filled draft creation form.

2. Stories Copy Hub

Manage and platform-tag all broadcast stories copy. Stories are grouped dynamically by their transmission (TX) date.

  • Active Broadcast Week: The system automatically highlights the date header matching the next upcoming Sunday in emerald green as "Active".
  • Unlinked / TBC Drafts: Leaving the date blank when creating/editing a story lists it in the TBC section pinned at the absolute top of the list.
  • Platform Toggles: Click the small platform badges (X, BS, IG, FB, TT, YT) next to any copy version to instantly flag which platforms that text version is approved or used for.
  • Copy Version Actions: Click the clipboard icon to copy the text to your clipboard. Click the pencil icon to push the copy straight into your scheduled drafts list.

3. Importing & Merging Files

  • Import DOCX (Master Copy): Click to upload your TEASER COPY 2026 MASTER document. It will automatically parse all TX dates, titles, legal notes, and copy versions. Re-importing updates text changes and preserves your existing platform toggle selections.
  • Import Analytics CSV: Drag-and-drop platform exports into the dashed box under the Scheduler to load real-world follower data and unlock "My Analytics".

Data Privacy & Browser Storage

By default, all dashboard changes, toggled platforms, and file imports are stored locally in your browser's Cache/LocalStorage. Clearing your cookies or website data will reset the dashboard to the default seed stories.

Cloud Database & Realtime Sync Setup (Firebase)

To sync copy across multiple users in real-time using Firebase:

  1. Create a new project at console.firebase.google.com.
  2. Go to Authentication -> Sign-in method, and enable **Email/Password**.
  3. Go to Firestore Database and click **Create database**. Set rules to allow read/write access (see below).
  4. In Project Settings, register a Web App and copy the config JSON block.
  5. Click the connection badge (labeled Local Mode or Offline) in the top header, paste the JSON, and connect.
Required Firestore Security Rules:
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /stories/{storyId} {
      allow read, write: if request.auth != null;
    }
  }
}
How Users Log In:

Once connected to Firebase, users are prompted to log in using their email and password. To register new users, click Sign Up on the login screen, enter their credentials, and they can sign in to participate in the same live workspace. You can also manually add users directly in the **Authentication** -> **Users** tab of your Firebase Console.
